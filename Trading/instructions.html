⚡ CYBER TRADER · 项目说明书

一个基于 Web 的量化交易模拟与策略开发平台 — 融合实时模拟、回测、虚拟交易员竞技、加密货币实盘数据。

1. 项目概述

CYBER TRADER 完全运行于浏览器前端，无需后端。用户可在 策略编辑器 中编写类似 Pine Script 的 JavaScript 策略， 并在 主模拟器 中观察策略在增强 GBM 市场模型下的表现，或切换至 加密货币实盘模式 接入币安实时 K 线。 回测模块 支持 Monte Carlo、Walk‑Forward 及多维参数扫描，虚拟交易员 功能可创建多个独立 AI 机器人同场竞技。

2. 主要功能

📈 实时模拟交易

增强 GBM（情景、天鹅、支撑压力），200ms tick，手动/策略自动交易。

🔗 加密货币实盘

币安 WebSocket + REST，1m/5m/15m K 线驱动，成交量归一化。

📝 策略编辑器

代码高亮、语法检查、自动补全、示例加载，一键应用至模拟器。

📊 回测模块

SIM(Monte Carlo)、CRYPTO(历史K线)、MULTI(μ-σ热力图)。

🤖 虚拟交易员

多独立机器人，独立资金/策略，排行榜+净值曲线。

📉 统计可视化

六边形雷达图、夏普、最大回撤、盈亏比、增量统计。

3. 模块详解

3.1 市场模型引擎 — MarketEngine.js & 子引擎

graph LR subgraph MarketEngine direction TB R[RegimeEngine 情景] --> C{stepPriceCore} S[SwanEngine 天鹅] --> C SR[SREngine 支撑压力] --> C C --> P[newPrice & 成交量] end style MarketEngine fill:#0a1a2a,stroke:#00f5ff

图2 · 市场引擎内部结构

设计目标： 提供比简单随机游走更逼真的价格路径，包含趋势、震荡、突变。

  • RegimeEngine：管理 BULL/BEAR/CHOP/V.BULL/Q.BEAR 五种情景，情景间平滑过渡，参数加入随机噪声。
  • SwanEngine：拟周期函数产生 ±2/±3 级天鹅事件，影响波动率、漂移和成交量。
  • SREngine：基于三条 EMA (20/55/120) 的动态支撑/压力，近区回归力 + 突破动量，自适应强度。
  • 增强跳跃：跳跃概率含情景乘数 + 全局噪声 + 聚类效应，大小与波动率正相关，熊市偏负。
  • 行为动量反转：连续单向运动累积，超过阈值后施加反向修正力。

关键数据结构

  • REGIMES 数组：五类情景基准参数
  • _currSnap, _prevSnap：带噪声的情景快照
  • swanLevel, swanTOffset
  • srEmaValues, srPrevSide, srTouchCount, srBreakDrift (Maps)

3.2 策略执行引擎 — StrategyEngine.js

sequenceDiagram participant Main as 主循环 (tick) participant SE as StrategyEngine participant Env as pineEnv participant Strat as 用户策略 Main->>SE: tick() SE->>Env: 构建环境(价格/指标) SE->>Strat: fn.call(ctx, env) Strat->>Env: 调用指标函数 Strat->>SE: strategy.entry/add/close SE->>Main: placeOrder/closeOrder (通过deps) Main-->>SE: onAfterTick (refreshUI)

图3 · 策略执行时序

设计目标： 安全运行用户编写的 JavaScript 策略，提供 Pine Script 风格 API。

  • start(code) 编译代码，保存函数。
  • 每 tick 调用 tick()，构造 pineEnv (含 open/high/low/close/volume、指标函数、strategy 对象)。
  • crossover/crossunder 通过 _crossState 跨 tick 记忆。
  • 策略中可使用 this.xxx 持久化变量。
  • 错误捕获，连续错误超过 5 次自动停止策略。

3.3 回测模块 — BackTest.js

graph TB subgraph BackTest A[选择模式] --> B{SIM} A --> C{CRYPTO} A --> D{MULTI} B --> E[生成GBM路径] --> F[_runStrategy] --> G[统计] C --> H[拉取币安K线] --> I[Walk-Forward分段] --> F D --> J[μ-σ 40x40网格] --> K[每格多次SIM] --> L[热力图] end

图4 · 回测流程

  • SIM 模式：自包含 GBM 模拟器 (_simMarket)，支持情景和随机种子，Monte Carlo 多次运行。
  • CRYPTO 模式：从币安获取历史 K 线，单次或 Walk‑Forward 分段测试。
  • MULTI 模式：遍历 μ[-0.4,0.4] 和 σ[0.025,1.0] 网格，每格多次回测取均值，生成总收益/胜率/盈亏比热力图。
  • _runStrategy 是独立于主模拟器的单次回测循环，维护现金/订单，执行策略代码。
  • 统计计算包括总收益、胜率、最大回撤、夏普、盈亏比、手续费拖累、换手率等。

3.4 加密货币实况模块 — Crypto.js

graph LR subgraph CryptoMode direction TB Enter[点击CRYPTO] --> Fetch[拉取500根历史K线] Fetch --> WS[连接WebSocket] WS --> Msg{收到消息} Msg -- k.x=false (实时) --> Live[更新最后K线高/低/收] Msg -- k.x=true (关闭) --> Close[完整游戏步进] Close --> CheckSLTP & Strategy & UI end

图5 · 加密货币模式数据流

通过 _cryptoCtx 对象与主模拟器状态绑定，替代内部 tick 驱动。

  • 历史成交量归一化（除以中位数），使成交量显示平滑。
  • 实时 K 线更新期间，图表每帧重绘，但策略和风控仅在 K 线关闭时触发。
  • 支持多交易对（BTCUSDT, ETHUSDT...）和时间周期 (1m/5m/15m)。

3.5 EMA与聚合缓存 — EMACache.js

graph TB subgraph Cache A[每 tick] --> B[updateEMA 增量/全量] B --> C[emaCache] subgraph Agg D[每帧 getAggregated] --> E{_aggCache有效?} E -- 否 --> F[重建聚合K线 O(N)] E -- 是 --> G[仅更新partial candle] end H[getAggEMA] --> I[稳定段缓存 aggEmaCache] I --> J[拼接partial EMA] end

图6 · 缓存机制

  • EMA缓存：对每个周期维护 Float64Array，支持增量左移复用，避免每帧重算。
  • 聚合K线缓存：将原始 ticks 按 candleSize 分组，稳定段一次重建，partial candle 每帧插值更新。
  • 聚合EMA：稳定段 EMA 缓存，最后一根用 O(1) 递推，复用 scratch buffer 无分配。

3.6 虚拟交易员 — VirtualTraders.js

graph LR subgraph VirtualTraders A[交易员列表] --> B[每tick遍历] B --> C[每个运行中机器人] C --> D[SL/TP/爆仓检查] C --> E[执行策略(独立订单/统计)] C --> F[更新净值历史 & MDD] B --> G[缓存指标 _piCache] end H[UI 卡片/排行榜] --> A

图7 · 虚拟交易员内部

  • 每个交易员拥有独立 orders, history, stats, equityHist, _stratCtx。
  • 策略代码独立编译执行，通过闭包 API 操作自身账户。
  • 指标缓存 _piCache 避免同 tick 重复计算。
  • UI 卡片采用增量更新，不重建 DOM，只改数值和类；仅 add/remove 时重建网格。
  • 排行榜按净收益排序，含收益百分比、胜率、性能条。

3.7 主模拟器界面 — TradingSimulator.html

核心 UI 容器，整合所有模块。主要组件：

  • HUD（净资产、盈亏、杠杆、现货价、现金、情景）
  • Canvas 图表（折线/蜡烛，EMA，成交量，开仓标记）
  • 工具栏（TF、图表模式、EMA开关、重置、设置）
  • 控制面板（LONG/SHORT、仓位大小、杠杆、止损/止盈）
  • 策略面板（代码文本域、启动/停止、编辑器按钮）
  • 订单面板（持仓卡片、历史、摘要）
  • 历史面板（增量渲染交易记录）
  • 摘要面板（净值曲线、雷达图、详细统计）
  • 内嵌编辑器模态框（StrategyEditor.html）

增量统计：_pushTrade 和 _recordNetAssetStat 在每笔交易和每 tick 更新统计变量，避免遍历历史数组。

3.8 策略编辑器 — StrategyEditor.html

  • 代码编辑区 + 透明错误覆盖层，语法检查分两阶段：纯语法检查（new Function）和运行时检查（eval + sourceURL）。
  • 错误行定位：逐行删除法定位语法错误行；堆栈解析提取引用错误行列。
  • 自动补全基于预定义关键词，支持 Tab 插入。
  • 示例加载 (MA Cross, RSI, Breakout, Persist, EMA Roll)。
  • 通过 window.parent.postMessage 将代码发送给主模拟器。
  • “BACKTEST”按钮读取父窗口参数，调用 BackTest.open。

5. 数据存储

纯内存 所有数据仅存在于运行时 JavaScript 变量中，页面刷新即丢失。包括：

  • 交易历史 (tradeHistory)
  • 策略代码 (textarea 初始值)
  • 回测结果 (面板内)
  • 虚拟交易员状态 (_traders 数组)

适合即时实验，无持久化。用户需自行复制代码或截图保存结果。

6. 界面交互要点

🎮 实时反馈

开平仓闪光，止损止盈 toast 通知，爆仓红色闪光+警示文字。

📱 响应式设计

移动端适配，增大触摸目标，调整布局为单列。

🖱️ 滑块/输入联动

仓位大小、止损、止盈可通过滑块（百分比）或直接输入价格控制，两者互斥。

📂 折叠面板

订单面板可折叠，历史和摘要面板通过按钮切换显示。

⚠️ 错误提示

策略语法错误在编辑器中高亮；SL/TP 不合理时输入框变红并显示错误信息。

7. 总结

CYBER TRADER 将复杂的市场模拟、策略执行、回测评估和虚拟竞技整合于单一前端，模块间通过清晰 API 通信，数据流简洁，缓存机制保证 60fps 流畅度。其设计亮点：

  • 多层次市场模型增强真实感；
  • 独立策略引擎与虚拟交易员实现多策略并行；
  • 专业级回测模块支持多种数据源和网格扫描；
  • 完全前端实现，易于部署和分享。

适用于量化爱好者、策略开发者进行快速实验和验证。
